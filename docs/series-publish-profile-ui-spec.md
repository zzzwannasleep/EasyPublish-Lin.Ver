- `版本`
- `发布配置`
- `站点稿`

不建议继续把面向用户的主文案写成：

- `模板`
- `矩阵`
- `套用模板默认值`

- `套用模板默认值` -> `选择发布配置`

## 5. 总体设计原则

### 5.1 先配置，再执行

剧项目主页的主心智应当是：

1. 先定义这部剧平时如何发布
2. 再把这套配置应用到某一集
3. 再进入本集执行和发布

不允许页面一打开就把用户直接推入超长版本表单。

### 5.2 桌面工作台优先

该界面是桌面端高频工具，不是网页后台。

必须满足：

- 信息密度高
- 常用操作常驻可见
- 大屏宽度被充分利用
- 不依赖连续弹窗串联主流程

### 5.3 先结构一致，再视觉打磨

第一轮 UI 改造优先保证：

- 分区清晰
- 术语统一
- 交互顺序正确
- 状态可理解

不要求第一轮就把颜色、阴影、圆角、动效打磨到最终状态。

## 6. 视觉与样式约束

### 6.1 视觉基调

剧集工作台必须继承全局桌面工作台方向，不另起一套轻飘的页面风格。

视觉关键词：

- 深色桌面工作台
- 铜橙强调
- 克制的成功 / 警告 / 失败色
- 强信息层级
- 不是默认 Element Plus 裸样式

基础方向沿用 [easypublish-redesign-plan.md](/c:/Users/65282/Downloads/PT自动发布脚本/docs/easypublish-redesign-plan.md) 中的约束：

- 背景深石墨
- 主面板深灰
- 强调色偏铜橙
- 状态色低饱和

### 6.2 面板层级

只允许使用两层主要面板层级：

1. `主工作面板`
2. `次级信息面板`

建议约束如下：

- `主工作面板`
  - 用于发布配置编辑区、集执行区
  - 视觉对比更强
  - 承载输入、保存、切换
- `次级信息面板`
  - 用于项目摘要、版本状态、站点权限提示
  - 视觉对比略弱
  - 承载说明、统计、只读信息

不建议同一屏里出现三四层乱序卡片视觉层级。

### 6.3 布局密度

推荐桌面布局密度：

- 卡片内边距：`16px` 到 `20px`
- 卡片间距：`12px` 到 `16px`
- 区块标题与正文间距：`8px`
- 表单列间距：`12px`

不要为了“看起来宽松”把桌面工具做成移动端式大留白页面。

### 6.4 按钮层级

每个主要区块同一时刻只允许一个主按钮。

推荐规则：

- `主按钮`
  - 保存发布配置
  - 为当前集应用配置
  - 进入当前版本编辑
- `次按钮`
  - 复制配置
  - 另存为
  - 从上一集继承
- `危险按钮`
  - 删除配置
  - 删除版本

不允许在同一行塞多个视觉上同权重的主按钮。

### 6.5 标签与状态胶囊

以下信息优先用 `tag` 或轻量状态胶囊展示：

- 默认配置
- 当前启用配置
- 已发布
- 待补发
- 未授权站点
- 必填字段未完成

以下信息不应只埋在说明文字里：

- 当前配置是否未保存
- 当前站点是否没有发布权限
- 当前版本是否沿用了某个发布配置

### 6.6 表单样式

表单必须遵守：

- 标签在上，输入在下
- 宽屏优先双列或分组布局
- 说明文案简短，避免段落式说明挤占编辑区
- 必填项有明确标识
- 禁用项必须说明禁用原因

不建议继续维持一整列从上到下的长表单。

## 7. 页面结构规范

### 7.1 剧项目主页整体结构

剧项目主页不建议做成纯三竖栏。

更推荐的是 `顶部摘要 + 横向发布配置卡片带 + 中部混合工作区`：

- 顶部：项目摘要与高频动作
- 第二层：横向发布配置卡片带
- 第三层左侧：当前发布配置编辑区
- 第三层右侧：集与执行侧栏

也就是说，主视觉重心应当是：

1. 先横向浏览常用发布配置
2. 再进入当前配置的编辑卡片区
3. 右侧始终保留当前集执行信息

当窗口宽度不足时，退化为：

- 上方：项目摘要
- 下方：横向发布配置卡片带
- 再下方：当前发布配置编辑区
- 最下方：集与执行区

无论如何退化，都不改变信息顺序。

### 7.2 顶部固定摘要区

页面顶部应固定一条摘要区，至少展示：

- 剧项目名称
- 季度 / 制作组摘要
- 默认发布配置
- 集数 / 版本数 / 待补发站点数
- 最近更新时间

摘要区右侧动作建议只保留：

- `新建发布配置`
- `新增一集`
- `打开项目目录`

不要把低频操作都堆到顶部命令栏。

### 7.3 发布配置卡片带

发布配置不一定非要占据一整列左栏。

更推荐的形式是横向卡片带，必要时支持左右滚动或折叠查看更多。

每个配置卡片建议展示：

- 配置名称
- 默认标记
- 启用站点数量
- 默认版本规格摘要
- 最近更新时间

每张卡片最多允许一个快捷动作，例如：

- 复制
- 删除

不要在卡片上堆满操作按钮。

卡片带负责“快速切换和扫描”，不负责承载完整编辑表单。

### 7.4 中部发布配置编辑区

编辑区必须固定为以下版块顺序：

1. `基础信息`
2. `标题预设`
3. `站点稿模板`
4. `默认发布站点`
5. `站点默认字段`
6. `保存操作栏`

不允许把“默认站点”和“站点字段”拆得太远。

因为用户必须先知道：

- 我发哪些站
- 这些站分别还要填什么

中部编辑区建议使用“纵向卡片堆叠”，而不是一整张无限长表单。

也就是说：

- 一个卡片 = 一个配置模块
- 卡片内部再做双列或分组布局
- 卡片之间通过标题和留白区分层级

### 7.5 右侧集与执行区

右侧执行区建议固定为以下顺序：

1. `集列表`
2. `当前集摘要`
3. `选择发布配置`
4. `本集差异字段`
5. `版本列表`
6. `进入编辑 / 检查 / 发布`

这里必须明确：

- 配置区解决“以后怎么发”
- 执行区解决“这一集现在怎么发”

右侧区可以是竖向侧栏，因为它承担的是“当前集上下文”和“快捷动作”，信息块天然更短。

也就是说，推荐的结构是：

- 发布配置：横向卡片带
- 配置编辑：中部卡片堆叠
- 集执行：右侧短信息侧栏

而不是所有部分都做成一排排竖列。

### 7.6 新建下一集的推荐流程

新增一集成功后，应立即进入轻量引导，而不是让用户自己再找入口。

推荐顺序：

1. 创建集成功
2. 弹出 `选择发布配置` 面板
3. 提供三个入口
4. 自动生成当前集初始版本

三个入口建议固定为：

- `套用默认配置`
- `从上一集继承`
- `从空白开始`

推荐默认高亮第一项，但如果上一集刚发布完成，也可以优先推荐第二项。

## 8. 模块级交互规则

### 8.1 发布配置切换

切换发布配置时：

- 如果当前编辑区没有未保存改动，直接切换
- 如果存在未保存改动，必须弹出确认

确认弹层文案应明确说明：

- 当前配置尚未保存
- 是否保存后切换
- 是否放弃改动

推荐操作顺序：

- `保存并切换`
- `放弃改动`
- `取消`

### 8.2 发布配置保存

每个配置编辑区底部应有固定保存栏。

保存栏至少包含：

- `保存`
- `另存为`
- `复制配置`

当存在未保存改动时，应出现明显但克制的状态提示，例如：

- `有未保存改动`

不要把保存动作藏在页面角落。

### 8.3 标题预设编辑

标题预设区应同时包含：

- 模板输入
- 可用变量列表
- 实时预览

实时预览必须始终可见，不允许让用户一边编辑一边猜结果。

如果缺少关键变量值：

- 预览区域显示占位态
- 不直接报错
- 在保存或应用时再做字段校验

### 8.4 站点稿模板编辑

站点稿模板区建议按站点卡片展示，而不是单个超长文本区。

每张卡片建议包含：

- 站点名
- 是否启用
- 是否沿用全局标题
- 标题覆盖模板
- 正文模板
- 站点说明

如果站点未启用：

- 卡片保持折叠或弱化态
- 不显示详细字段

### 8.5 默认发布站点选择

站点选择必须区分三种状态：

1. `可发布`
2. `已配置但当前无权限`
3. `未配置账号`

展示要求：

- 可发布：正常勾选
- 已配置但无权限：允许显示但不可编辑，并提示原因
- 未配置账号：显示引导去账号配置

不能只用“灰掉”来表达所有异常状态。

### 8.6 站点默认字段区

站点默认字段区只显示：

- 已勾选站点
- 且该站点当前可发布

字段应按站点能力 schema 渲染，并显式区分：

- 必填
- 选填
- 只读
- 当前由项目配置继承

如果某站点缺少必填默认字段：

- 该站点卡片顶部显示警示
- 当前配置允许暂存
- 但在“应用到当前集”时必须二次提示

### 8.7 本集差异字段

本集差异字段区只应该出现与“这一集变化值”有关的内容，例如：

- 集号
- 分集标题
- 本集资源路径 / 种子
- 临时站点覆盖字段

不应把项目级长期稳定配置重新重复一遍。

### 8.8 版本列表

版本列表必须优先回答这几个问题：

- 这一集有哪些版本
- 每个版本用了哪套配置
- 每个版本还差哪些站
- 哪个版本正在编辑

每个版本卡片建议展示：

- 版本名称
- 关联发布配置名
- 标题预览
- 目标站点数量
- 已发布 / 待补发 / 失败数量
- 最近更新时间

主动作建议只保留：

- `载入编辑`
- `回写版本`

低频动作可以放到二级菜单。

## 9. 状态规范

### 9.1 空状态

必须定义的空状态包括：

- 没有任何发布配置
- 没有任何集
- 当前集没有任何版本
- 当前站点没有可编辑字段

空状态文案必须同时包含：

- 当前为什么是空的
- 下一步应该做什么

例如：

- `还没有发布配置，先创建一个常用方案。`
- `还没有任何集，先新增第 01 集。`

### 9.2 加载状态

加载态应优先使用骨架屏或区块级 loading，不建议整页白屏等待。

推荐规则：

- 首次进入页面：允许区块骨架
- 切换配置：只锁当前编辑区
- 新增一集：只锁相关动作按钮

### 9.3 禁用状态

禁用控件必须说明原因。

至少覆盖以下场景：

- 没有站点账号
- 站点没有发布权限
- 当前集未选择
- 当前配置未保存，暂不能应用

禁止只把按钮置灰而没有任何提示。

### 9.4 错误状态

错误信息分两层：

- `区块错误`
  - 例如配置保存失败、站点字段加载失败
- `页面级错误`
  - 例如剧项目读取失败

区块错误应就地展示，页面级错误才允许占据主区域。

### 9.5 成功状态

成功反馈以轻提示和局部状态变化为主。

例如：

- 保存配置成功
- 当前集已应用配置
- 已从上一集继承版本

不要每个小成功都弹强打断式对话框。

## 10. 样式细则

### 10.1 信息层级

标题层级建议固定为三层：

- 页面标题
- 区块标题
- 卡片标题

不要在同一屏里混用过多字号。

### 10.2 文案长度

说明文案应短，不写成长段帮助文档。

推荐规则：

- 区块说明：一行到两行
- 字段帮助：一句话
- 空状态：一句原因 + 一句引导

更长的说明应通过：

- 帮助链接
- Tooltip
- 折叠说明

承载，而不是常驻大段文字。

### 10.3 颜色使用约束

颜色只能承担“层级”和“状态”两种职责。

不要把颜色当作装饰。

推荐规则：

- 铜橙：当前主动作、当前焦点
- 绿色：成功、已覆盖
- 黄色：待补发、待完善
- 红色：失败、阻塞
- 灰色：次级说明、禁用项

### 10.4 图标使用约束

图标只用于增强扫描效率。

建议使用场景：

- 状态
- 站点
- 时间
- 目录

不建议每个按钮都强塞图标。

## 11. 字段优先级与覆盖规则

创建版本或应用配置时，字段优先级应明确为：

1. `项目默认值`
2. `发布配置默认值`
3. `当前集差异值`
4. `当前版本手动覆盖值`

页面必须在需要时提示“当前值来自哪里”。

推荐展示方式：

- `来自发布配置`
- `已被本集覆盖`
- `已被当前版本覆盖`

如果不展示来源，后续一定会让用户困惑为什么某个站点字段变了。

## 12. 组件映射建议

为了尽量复用现有技术栈，第一轮可优先采用：

- 列表：`el-menu` / 自定义列表卡片
- 主面板：自定义工作台卡片
- 状态：`el-tag`
- 提示：`el-alert`
- 表单：`el-form` + 分组卡片
- 低频动作：`el-dropdown`
- 确认：`el-dialog`

不建议第一轮引入大量新组件库。

## 13. 第一轮实现验收

如果第一轮 UI 改造完成，应至少满足以下验收：

1. 用户在剧项目主页能直接看到“发布配置”而不是“模板矩阵”
2. 页面能清楚区分“配置区”和“执行区”
3. 已选且可发布的站点，才会出现可编辑字段
4. 新增一集后，能立即选择配置而不是重新手动找入口
5. 版本列表能清楚展示“关联配置”和“待补发状态”
6. 整体界面仍保持桌面工作台风格，不退回默认组件库表单页
