### 4.2 这张草图要表达的重点

这张总览图有四个强约束：

1. `发布配置` 应先以横向卡片带呈现，而不是默认做成整列导航
2. `当前发布配置编辑区` 必须位于视觉中心
3. `集与执行区` 可以保留为右侧短侧栏
4. 顶部摘要区只放高频主动作，不堆低频按钮

## 5. 横向配置卡片带草图

### 5.1 发布配置卡片带

```text
+----------------------------------------------------------------------------------------------+
| 发布配置卡片带                                                                               |
| 说明: 先选一套常用发布方案，再修改细节                                                       |
|----------------------------------------------------------------------------------------------|
| [1080p 简中常规版 {默认}]   [2160p 双语完整版]   [Web 4K 轻量版]   [新建发布配置]            |
| 4个站点 · 1080p / 简中      2个站点 · 2160p/双语  1个站点 · Web/双语                         |
| 最近更新 12:20              最近更新 11:48         最近更新 10:31                             |
+----------------------------------------------------------------------------------------------+
```

### 5.2 卡片规则

每个卡片最多展示：

- 名称
- 默认标识
- 配置摘要
- 最近更新时间
- 一个二级操作入口

不要把下面这些信息全塞进去：

- 每站点字段详情
- 标题模板全文
- 正文模板全文
- 多个并列按钮

卡片带负责快速扫描和切换，不负责编辑。

### 5.3 为什么用横向卡片带

原因很直接：

- 常用发布配置天然适合横向比较
- 名称、站点数、规格摘要都属于“扫一眼”信息
- 顶部横向带能先建立全局感，再进入下方细编

如果继续做成高而窄的左栏，信息会更像文件树，不像“可复用发布方案”。

### 6.2 中栏规则

中部编辑区必须始终满足：

- 所有编辑都围绕一个当前发布配置展开
- 标题预设必须有实时预览
- 站点稿模板和站点字段必须就近出现
- 保存栏必须固定在底部或始终可见

中部推荐明确做成“卡片堆叠”而不是单大面板：

- 基础信息卡片
- 标题预设卡片
- 站点稿模板卡片
- 默认站点卡片
- 站点字段卡片
- 保存栏卡片

### 6.3 中栏滚动规则

推荐规则：

- 左栏独立滚动
- 中栏独立滚动
- 右栏独立滚动

不要让整个页面只有一个总滚动条。

原因很简单：

- 用户在改配置时，需要保留左栏选中态
- 用户在看右栏集状态时，不应把中栏编辑位置冲掉

### 7.2 右侧侧栏规则

右侧区之所以仍然适合做侧栏，是因为它承载的是短信息块和快捷动作。

右侧区只承载“当前集”的事情，不应该再次出现完整的项目级配置编辑表单。

右侧区应该始终回答这四个问题：

1. 当前是哪一集
2. 当前用了哪套发布配置
3. 当前集还有哪些差异值没填
4. 当前集各版本发布到什么程度

## 8. 新建下一集弹层草图

### 8.1 弹层结构

```text
+------------------------------------------------------------------+
| 新建第 06 集成功                                                 |
|------------------------------------------------------------------|
| 选择这集要套用的发布方式                                         |
|                                                                  |
| [推荐] 套用默认配置                                              |
| 1080p 简中常规版                                                 |
| 自动带出标题规则、站点稿、默认站点和站点字段                     |
| [立即套用]                                                       |
|------------------------------------------------------------------|
| 从上一集继承                                                     |
| 复制第 05 集的版本组合和大部分配置                               |
| 适合本集发布计划与上一集基本一致的情况                           |
| (从上一集继承)                                                   |
|------------------------------------------------------------------|
| 从空白开始                                                       |
| 不带任何发布配置，手动新建当前集版本                             |
| (从空白开始)                                                     |
|------------------------------------------------------------------|
| (稍后再说)                                                       |
+------------------------------------------------------------------+
```

### 8.2 弹层规则

这不是一个“复杂向导”，而是一个轻量决策弹层。

必须满足：

- 一屏看完
- 三个选项差异清楚
- 默认推荐路径明显
- 可跳过，但不建议默认跳过

不建议在这个步骤里塞入太多字段。

### 8.3 弹层后的系统行为

如果用户选择：

- `套用默认配置`
  - 自动生成当前集默认版本
  - 自动带入当前配置默认站点和字段
- `从上一集继承`
  - 复制上一集版本组合
  - 带入上一集使用的发布配置引用和快照
- `从空白开始`
  - 只创建当前集壳
  - 不生成版本

## 10. 窄屏退化草图

### 10.1 中等宽度下的堆叠顺序

当混合布局放不下时，推荐退化为：

```text
+--------------------------------------------------------------+
| 顶部摘要区                                                   |
+--------------------------------------------------------------+
| 发布配置卡片带                                               |
+--------------------------------------------------------------+
| 当前发布配置编辑区                                           |
+--------------------------------------------------------------+
| 集与执行区                                                   |
+--------------------------------------------------------------+
```

退化后仍必须保持：

1. 先配置卡片带
2. 再配置编辑
3. 再集执行

不允许把执行区提到配置区前面。

### 10.2 手机式单列不是目标

该页面虽然要能在较窄窗口下工作，但目标仍然是桌面端。

不建议把它做成：

- 超长单列表单
- 层层点进去的移动端交互
- 大量抽屉替代主界面结构

## 11. 第一轮实现建议

如果只做第一轮低风险实现，建议先达到下面这版线框：

### 11.1 第一轮必须有

- 顶部摘要区
- 左侧发布配置列表
- 中间配置编辑区
- 右侧集与执行区
- 新建下一集后的三选一弹层
- 版本列表中的“来源配置”展示

### 11.2 第一轮可以先简化

- 站点稿模板可以先只支持少量文本区
- 站点默认字段可以先只渲染少数已适配站点
- 标题预设可以先用模板文本 + 变量 chips，而不是复杂拖拽拼装
- 版本列表可以先用卡片，不一定先做复杂矩阵表

## 12. 和当前实现的差异清单

当前页面更接近：

- 先选集
- 先建版本
- 再看模板和默认值

目标页面应变成：

- 先看这部剧有哪些发布配置
- 先修改或选择发布配置
- 再把配置应用到当前集
- 再查看本集版本和发布动作

也就是说，最大变化不是“多几个字段”，而是主视觉重心前移到发布配置。
